## ****TiDB　分散DB導入****

以下オーロラで懸念があったため

- 高負荷時のwriteスループット
- クラスタサイズリミット、テーブルサイズリミット

### 良かったこと

- 水平スケールによるキャパシティ拡張
- 更新系パフォーマンス改善
- インスタンス障害時の影響軽減

### 改善点懸念点

- 参照系パフォーマンスがわずかに悪化
- 運用面とコスト面から物理的な参照ワークロードの分散がしづらい
- 無制限にできるスケールアウトによるコストコントロールの難しさ

## マルチリージョン、マルチクラスタに

- 大阪リージョンにレプリケーションするようにした。
